# 11. 프로젝트 구조

저장소 레이아웃과 코드 배치 규칙. **"내 코드는 어디에 둬야 해?"**에 대한 한 곳짜리 답.

데이터 흐름·기술 스택은 [01-아키텍처.md](01-아키텍처.md), 도메인 모델은 [03-데이터-모델.md](03-데이터-모델.md) 참고.

## 한눈에 보기

```
무임하차/
├── docs/         # 설계 문서 — 변하지 않을(또는 신중하게 변하는) 명세
├── design/       # 디자인 시안 — HTML 목업, 와이어프레임
├── server/       # NestJS 백엔드 (TypeScript)
├── client/       # 웹 클라이언트 (Vite + React + TS) — 데스크탑(Electron)은 V2
├── .github/      # PR·이슈 템플릿
├── README.md     # 진입점
├── CLAUDE.md     # AI 보조 작업 규칙
└── LICENSE       # MIT
```

서버와 클라이언트는 **별도 npm 프로젝트**로 분리되어 있다 (각자 `package.json`). 워크스페이스나 Nx 같은 모노레포 도구는 사용하지 않는다 — 학생 팀 4명 규모에 오버킬.

## 디렉터리별 책임

### `docs/` — 설계 문서

| 종류 | 들어가는 것 | 들어가지 않는 것 |
|---|---|---|
| 명세 | 기능, API, 데이터 모델, 인프라 제약 | 일일 작업 로그 |
| 결정 | 확정된 정책 (예: NestJS 채택) | 결정 진행 중 노트 |
| 참조 | 외부 자료 요약 (예: AWS 제약) | 외부 자료 사본 |

파일명은 `NN-한국어-제목.md` (두 자리 prefix로 정렬). 새 문서를 추가하면 [docs/README.md](README.md)의 TOC와 [루트 README](../README.md)의 문서 표에 둘 다 행을 추가한다.

### `design/` — 디자인 시안

HTML 목업·와이어프레임·디자인 토큰 시안. 현재는 `design/무임하차.html` 하나(메인 대시보드 시안). 디자인 시스템이 정립되면 `client/` 안으로 일부 이동될 수 있음.

### `server/` — NestJS 백엔드

```
server/
├── src/
│   ├── main.ts                  # bootstrap, 포트 listen
│   ├── app.module.ts            # 루트 모듈
│   ├── app.controller.ts        # 헬스 체크
│   ├── app.service.ts
│   └── <feature>/               # 도메인별 모듈 (구현 단계에서 추가)
│       ├── <feature>.module.ts
│       ├── <feature>.controller.ts
│       ├── <feature>.service.ts
│       ├── dto/                 # 입출력 DTO
│       └── entities/            # DB 엔티티
├── test/                        # e2e 테스트
├── nest-cli.json
├── tsconfig.json
└── package.json
```

새 도메인 모듈 추가 절차:

1. `nest g module <name>` → `src/<name>/`에 모듈 생성
2. `<name>.controller.ts`(REST 라우트) + `<name>.service.ts`(비즈니스 로직) + `<name>.module.ts`
3. `src/app.module.ts`의 `imports`에 등록
4. DTO·엔티티는 같은 폴더 하위 `dto/`·`entities/`에 둔다 — 모듈 간 import는 명시적으로

스타일·테스트: `npm run lint`(ESLint+Prettier), `npm run test`. 자세한 명령은 [server/README.md](../server/README.md).

### `client/` — 웹 클라이언트 (Vite + React + TS)

```
client/
├── src/
│   ├── App.tsx              # 루트 컴포넌트
│   ├── main.tsx             # ReactDOM render 진입점 (메인 탭)
│   ├── companion.tsx        # 회의 중 보조 창 진입점 (window.open 대상)
│   ├── assets/              # 이미지, 아이콘
│   ├── components/          # 공용 UI 컴포넌트
│   ├── features/            # 도메인별 화면 (구현 단계에서 추가)
│   ├── stores/              # Zustand/React Query 스토어 (도입 시)
│   ├── lib/                 # 유틸, API 클라이언트, WebSocket 래퍼, Web Speech 래퍼
│   └── styles/              # 전역 스타일, Tailwind 설정
├── public/                  # 정적 자산 (Vite가 그대로 서빙)
├── test/                    # 테스트
├── index.html               # 메인 탭 HTML
├── companion.html           # 보조 창 HTML (별도 진입점)
├── vite.config.ts
└── package.json
```

웹앱이므로 모든 코드는 **브라우저 컨텍스트**에서 동작한다. 마이크·STT는 `getUserMedia`·Web Speech API(`SpeechRecognition`)를 직접 사용하고, 회의 중 보조 UI는 `window.open`으로 띄우는 **별도 브라우저 창**(`companion.html`)이다 — 메인 탭과는 WebSocket/`BroadcastChannel`로 상태를 공유한다.

> **데스크탑(Electron)은 V2**: `electron/main`·`preload`·`stt-sidecar`(RealtimeSTT)·`electron-builder` 등 네이티브 레이아웃은 V2에서 추가한다. 계획은 [07-Electron-구현.md](07-Electron-구현.md).

### `.github/` — GitHub 협업 자산

- `pull_request_template.md` — 한국어, Conventional Commits + 프로젝트 제약 체크리스트
- `ISSUE_TEMPLATE/bug_report.md` — 버그 리포트
- `ISSUE_TEMPLATE/feature_request.md` — 기능 제안, "음성 원본 미저장" 등 제약 체크 포함

PR/이슈 템플릿을 추가·수정하면 한국어 + 프로젝트 제약 체크리스트 유지.

## 코드 추가 시 의사결정 트리

```
새 코드를 어디에 둬야 하나?
│
├─ 회의 데이터를 다루는 백엔드 로직?           → server/src/<feature>/
├─ REST 엔드포인트 추가?                       → server/src/<feature>/<feature>.controller.ts
├─ DB 엔티티 추가?                              → server/src/<feature>/entities/
├─ 발화 텍스트 검증 등 도메인 규칙?            → server/src/<feature>/<feature>.service.ts
│
├─ 회의 중 UI?                                  → client/src/features/meeting/
├─ 회의 전후 대시보드 UI?                       → client/src/features/dashboard/
├─ 공용 컴포넌트 (버튼, 모달 등)?               → client/src/components/
├─ Web Speech API 인식·재시작 로직?            → client/src/lib/speech.ts
├─ WebSocket 클라이언트?                        → client/src/lib/ws.ts
├─ 보조 창(window.open) 제어·상태 공유?         → client/src/lib/companion.ts
│
├─ 설계 결정·명세?                              → docs/NN-제목.md (+ TOC 갱신)
├─ 디자인 시안?                                 → design/
```

## 명명 규칙

| 대상 | 규칙 | 예 |
|---|---|---|
| 디렉터리 | kebab-case | `client/src/features/meeting-room/` |
| TS/TSX 파일 | kebab-case (React 컴포넌트 파일도 동일) | `contribution-bar.tsx` |
| React 컴포넌트 | PascalCase | `ContributionBar` |
| 함수·변수 | camelCase | `calculateContribution` |
| 상수 | UPPER_SNAKE_CASE | `MAX_MEETING_DURATION_MIN` |
| 환경 변수 | UPPER_SNAKE_CASE | `JWT_SECRET`, `VITE_API_BASE_URL` (Vite는 `VITE_` 접두사만 렌더러 노출) |
| NestJS 모듈 | PascalCase 클래스 + kebab-case 파일 | `MeetingModule` in `meeting.module.ts` |
| DB 테이블·컬럼 | snake_case | `utterance_count` (TypeORM 네이밍 전략으로 매핑) |
| 브랜치 | `feature/<설명>`, `fix/<설명>` | `feature/stt-poc`, `fix/contribution-overflow` |
| 커밋 | Conventional Commits | `feat(stt): Web Speech API 인식 통합 추가` |

## 환경 변수

| 위치 | 파일 | 추적 여부 |
|---|---|---|
| 서버 | `server/.env` | gitignore — 절대 커밋 금지 |
| 서버 (템플릿) | `server/.env.sample` | 추적 — 키 목록만, 값은 빈칸 |
| 클라이언트 | `client/.env` | gitignore — 절대 커밋 금지 |
| 클라이언트 (템플릿) | `client/.env.sample` | 추적 — `VITE_` 접두사 키만 |

루트 `.gitignore`는 `.env*`를 차단하되 `!.env.sample`로 화이트리스트. 새 키를 추가하면 양쪽(`.env`와 `.env.sample`)에 모두 반영.

비밀 값(`JWT_SECRET`, `OPENAI_API_KEY`, `KAKAO_CLIENT_SECRET` 등)은 EC2에 SSH로 직접 작성하거나 SSM Parameter Store에서 주입. 자세한 운영 환경은 [10-AWS-인프라-제약.md](10-AWS-인프라-제약.md).

## 두지 않는 곳 (자주 헷갈리는 안티패턴)

| 하지 마라 | 대신 |
|---|---|
| 루트에 새 디렉터리 추가 (`utils/`, `common/` 등) | `server/`나 `client/`의 적절한 하위에 둔다 |
| 서버와 클라이언트 공용 코드를 루트로 추출 | 학생 팀 규모에서는 중복이 낫다. 정 필요하면 그때 다시 논의 |
| 음성 원본 파일을 어딘가에 저장 | **금지** — 우리 서버에는 텍스트만 전송 ([01-아키텍처.md](01-아키텍처.md) §음성 데이터 처리 원칙) |

## 산출물 디렉터리 (모두 gitignore)

| 경로 | 용도 |
|---|---|
| `server/dist/`, `server/node_modules/` | NestJS 빌드·의존성 |
| `client/dist/`, `client/node_modules/` | Vite 빌드·의존성 |
| `*/logs/` (`logs/`, `docs/logs/`, `design/logs/`, `server/logs/`, `client/logs/`) | AI 보조 도구 부산물 |
| `.claude/`, `.gstack/`, `.superpowers/` | AI 보조 도구 메타데이터 |
| `*.onnx`, `*.bin`, `*.wav`, `*.mp3`, `*.webm`, `models/`, `data/` | PoC 음성 산출물 (V2 RealtimeSTT 모델 포함) |

이들 중 어느 하나라도 `git status`에 보이면 `.gitignore` 누락 신호.
