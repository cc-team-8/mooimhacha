# 01. 아키텍처 & 기술 스택

## 데이터 흐름

```
[각 사용자의 웹 브라우저]
  마이크 → getUserMedia 오디오 캡처
        → 브라우저 내장 Web Speech API (SpeechRecognition)
            (음성은 브라우저 외부 Google 서버로 전송되어 텍스트로 변환됨 — MVP 한계)
        → 발화 단위(final result)로 텍스트만 우리 서버에 전송
              ↓
        [서버 — EC2 단일 인스턴스의 NestJS]
          ├─ MySQL(RDS)에 utterance 저장
          ├─ 인메모리 Map에 사용자별 누적 글자수 갱신
          │    └─ 5초마다 RDS에 flush
          └─ WebSocket broadcast (1초 디바운스)
              ↓
[모든 사용자의 웹 브라우저]
  Framer Motion으로 기여도 바 갱신
```

핵심: STT는 **브라우저 내장 Web Speech API**로 수행(MVP). 우리 서버로는 변환 완료된 **텍스트만** 전송·저장하고, 서버는 저장·집계·브로드캐스트만 담당한다. 단, Web Speech API는 음성을 브라우저 외부(Google) 서버로 보내므로 *우리 서버는 음성을 받지 않지만 음성이 기기를 완전히 떠나지 않는 것은 아니다* — 이 데이터 주권 한계는 V2의 RealtimeSTT 로컬 추론 전환으로 해소할 계획이다([05](05-STT-음성-처리.md)·[08](08-우선순위-로드맵.md)).

## 클라이언트 (웹 — 브라우저)

| 영역           | 기술                                                    |
| -------------- | ------------------------------------------------------- |
| 프레임워크     | React + TypeScript (Vite)                               |
| 배포 형태      | 브라우저 웹앱 (HTTPS 필수) — 데스크탑(Electron)은 V2 보류 |
| UI             | TailwindCSS + shadcn/ui                                 |
| 상태관리       | Zustand 또는 React Query                                |
| 애니메이션     | Framer Motion                                           |
| STT            | Web Speech API (`SpeechRecognition`) — 브라우저 내장     |
| 회의 중 보조 UI | 별도 브라우저 창 (`window.open`) — always-on-top은 V2     |
| 로컬 저장      | `localStorage` / IndexedDB (창 위치 등)                  |

> **PoC 메모 (2026-05-29)**: MVP STT 엔진은 **Web Speech API(브라우저 내장)** 로 채택. 데이터 주권 한계와 ~60초 강제 종료(자동 재시작으로 방어)를 수용한다. 로컬 추론(RealtimeSTT, faster-whisper)은 **V2**로 보류. 1주차 PoC에서 한국어 정확도·실시간 지연·재시작 안정성을 검증한다([05](05-STT-음성-처리.md)).

## 백엔드

| 영역        | 기술                                                            |
| ----------- | --------------------------------------------------------------- |
| 서버        | NestJS (Node.js)                                                |
| DB          | MySQL (RDS 프리티어)                                            |
| DB 접근     | TypeORM                                                         |
| 캐시        | NestJS 프로세스 인메모리 Map (5초마다 RDS flush) — Redis 미사용 |
| 실시간 통신 | WebSocket (NestJS Gateway 직접 서빙, 단일 EC2)                  |

## 외부 API

| 영역 | 서비스                          | 용도                      | 비고                                       |
| ---- | ------------------------------- | ------------------------- | ------------------------------------------ |
| STT  | Web Speech API (브라우저 내장)  | 음성 → 텍스트             | 무료, 별도 키 불필요. 음성이 브라우저 외부(Google) 서버로 전송됨(MVP 한계). V2에서 RealtimeSTT 로컬 추론으로 전환 예정 |
| LLM  | OpenAI GPT-4o-mini              | 회의 중 안건별 요약, 회의 후 종합 정리·다음 회의 안건 생성 | 회의 중 안건당 1회 + 회의 후 2회 |

## 인프라 (AWS)

**Start AWS 학생 활동 프로그램** 제약 하에서 운영. 상세는 [10. AWS 인프라 & 제약](10-AWS-인프라-제약.md) 참조.

| 영역 | 선택 |
| --- | --- |
| 컴퓨트 | EC2 1대 (t3.small), Docker로 NestJS 운영 |
| DB | RDS MySQL 프리티어 (퍼블릭 액세스 허용, EC2 보안 그룹에서만 3306 인입) |
| 캐시 | 별도 매니지드 캐시 없음 — NestJS 인메모리 |
| 도메인/HTTPS | **필수** — 웹앱의 `getUserMedia`·Web Speech API가 보안 컨텍스트(HTTPS)에서만 동작. 도메인 + TLS(예: Caddy + Let's Encrypt 리버스 프록시)로 정적 프론트 서빙 + `wss://` 프록시. 상세는 [10](10-AWS-인프라-제약.md) |
| 로깅 | EC2 stdout → CloudWatch Logs |
| 배포 | 수동 (`git pull && docker compose up -d --build`) — Access Key 발급 불가 |
| 리전 | ap-northeast-2 (서울). Bedrock 사용 시 us-east-1 |
| Role | EC2 인스턴스 프로필 `SafeInstanceProfileForUser-{username}` |

## 모드 분리

- **회의 중 모드**: 별도 브라우저 창(`window.open`, 폭 400px), 곁눈질로 한눈에 파악 가능한 디자인. always-on-top은 V2(데스크탑)에서 제공
- **회의 외 모드**: 메인 탭 풀 화면 대시보드

## 음성 데이터 처리 원칙

- 음성 파일은 어디에도 저장하지 않음 — 텍스트만 보관
- **우리 서버**로 전송·저장되는 것은 변환 완료된 텍스트뿐 (우리 인프라는 음성을 받지 않음)
- 단, MVP는 Web Speech API를 쓰므로 음성이 **브라우저 외부(Google) STT 서버로 전송**된다 — 데이터 주권 측면의 한계이며, V2에서 로컬 추론(RealtimeSTT)으로 전환해 음성이 기기를 떠나지 않도록 회복할 계획이다
